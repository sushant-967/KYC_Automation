# TCS & AMD AI Hackathon

Innovation challenge for TCS developers/engineers to build enterprise-ready AI solutions using modern AI frameworks on AMD-powered accelerated computing. Focus areas: GenAI, RAG, AI agents, vision, speech, multimodal.

## Key Dates

| Event | Date |
|---|---|
| Hackathon Announcement | 25 May 2026 |
| Registration Closes | **5 June 2026** |
| Virtual Kick-off | 8 June 2026 |
| Hands-on Workshop | 9 June 2026 |
| Build Phase | 8–12 June 2026 |
| Final Submissions Due | 12 June 2026 |

## Who Can Participate

- TCS developers and engineers across business groups
- Interest in AI/ML, data, cloud, or application development
- **Format:** Individual OR team (2–3 members); cross-functional teams encouraged but not required

## Tracks

### Track 1 — Agents
Single/multi-agent orchestration. RAG pipelines, agents, regression models, chatbots for BU-specific tasks. Open-source stacks (Llama, Deepseek, Qwen, Graph AI, predictive ML). Focus: productivity, efficiency (tokens, latency), accuracy.

### Track 2 — Multimodal
For manufacturing, networking, industrial, pharma. Combined pipelines (vision + reasoning + speech + video analysis) for end-to-end decision-making.

### Track 3 — Fine-Tuning (Advanced)
Apply PEFT, GRPO, etc. to adapt open-source LLMs for domain-specific tasks (code gen, telecom, pharma). Must show measurable accuracy/performance gains.

## Prerequisites

- Coding in Python, Java, Go, Rust, etc.
- Familiarity with TensorFlow, PyTorch, Scikit-learn, or similar
- Problem-solving mindset, real-world focus

## Steps

1. **Register** via Ultimatix Prime Hub (individual or team)
2. **Prepare** — review theme/rules, pick use case, attend virtual launch + workshop, sign up for AMD AI Developer Program, AMD Developer Cloud, Discord; set up dev environment on AMD Developer Cloud
3. **Build** during the hackathon window using approved tools
4. **Submit** PPT/PDF + demo video via Ultimatix Prime events
5. **Demo & Evaluation** — shortlisted teams present at grand finale

## Submission Requirements

3–5 slide PDF/PPT + demo recording.

| Slide | Content |
|---|---|
| 1. Basic Info | Team name, members & roles, use case detail, short description |
| 2. Problem & Context | Problem statement, target users, business relevance, mapped track |
| 3. Solution Overview | Architecture/workflow diagram, AI approach, frameworks, what was built |
| 4. Model Insights | Models, datasets (if fine-tuning), training time, tokens used (example scenarios), end-to-end latency, GPU usage + memory |
| 5. Impact & Demo Summary | Expected impact, differentiators, demo flow, planned future extensions |

## Evaluation Criteria

| Criteria | Weight |
|---|---|

## Guidelines

- All development must happen during the hackathon period
- Open-source libraries OK, but must declare what was built during the hackathon
- Prior code reuse must be disclosed
- Submission must include demo, presentation, code link, datasets used
- Plagiarism/rule violations → disqualification

- **Technical Implementation is 40%** — a working end-to-end solution beats a flashy idea
- **Innovation is only 15%** — don't over-index on novelty
- Slide 4 metrics (tokens, latency, GPU memory) → instrument from day 1; they double as Technical Implementation evidence
- 4-day build window → scope tight; Track 1 (Agents) is safest, Track 3 (Fine-Tuning) is risky without prior PEFT experience
- "Built during hackathon" rule → research/scaffolding plan ready by June 8, no actual code yet
